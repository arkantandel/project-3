## рҹ“– Overview

This project builds a **fault-tolerant, cloud-native data pipeline** using AWS services and Docker.

The pipeline does the following automatically:

> рҹ“Ҙ **Reads** a CSV file stored in **Amazon S3**  
> рҹ“Ө **Inserts** the data into **Amazon RDS (MySQL)** database  
> вҡ пёҸ **Falls back** to **AWS Glue Data Catalog** if RDS is unavailable  
> рҹҗі Everything runs inside a **Docker container** вҖ” portable and consistent

This simulates a real-world DevOps/Cloud scenario where data must always be ingested вҖ” even when the primary database is down.

## рҹӣ пёҸ Step-by-Step Setup Guide

### вң… Step 1 вҖ” Create an S3 Bucket and Upload CSV

```
AWS Console вҶ’ S3 вҶ’ Create Bucket
Bucket Name : my--data-pipeline-bucket
Region      : us-east-1
```

Upload `sample_data.csv` to the bucket.

```csv
id,name,age
1,Arkan,22
2,Rahul,25
3,Amit,28
```

> рҹ’Ў **Why?** S3 acts as the source of truth for our data. The Python app will read this file.

---

### вң… Step 2 вҖ” Launch RDS MySQL Instance

```
AWS Console вҶ’ RDS вҶ’ Create Database
Engine      : MySQL
Template    : Free Tier
DB Name     : testdb
Username    : admin
Password    : your-password
Public      : Yes (for testing)
```

Note down the **endpoint URL** вҖ” you'll need it later.

> рҹ’Ў **Why?** RDS is our primary destination. All CSV data will be inserted here as a table.

---

### вң… Step 3 вҖ” Create AWS Glue Database

```
AWS Console вҶ’ Glue вҶ’ Databases вҶ’ Add Database
Name : my_glue_db
```

> рҹ’Ў **Why?** Glue is the fallback. If RDS fails, the app creates a table here pointing to the S3 data.

---

### вң… Step 4 вҖ” Launch EC2 Ubuntu Instance

```
AWS Console вҶ’ EC2 вҶ’ Launch Instance
OS     : Ubuntu 22.04
Type   : t2.micro (Free Tier)
Key    : Create or use existing .pem key
```

SSH into the instance:
```bash
ssh -i your-key.pem ubuntu@your-ec2-public-ip
```

---

### вң… Step 5 вҖ” Install Docker on EC2

```bash
sudo apt update
sudo apt install docker.io -y
sudo usermod -aG docker ubuntu
newgrp docker
docker --version
```

> рҹ’Ў **Why?** Docker lets us run our app in an isolated container without worrying about the host environment.

---

```bash
git clone https://github.com/YOUR_USERNAME/aws-pipeline.git
cd aws-pipeline
```

## вҡЎ Challenges & Solutions

| # | Challenge | Solution |
|---|-----------|----------|
| 1 | Passing AWS credentials securely into Docker | Used `-e` environment variables at runtime. Nothing is hardcoded in the image. |
| 2 | RDS not accessible from Docker container | Opened port 3306 in the RDS Security Group for the EC2 instance IP |
| 3 | Glue fallback only triggering when needed | Wrapped RDS code in `try/except` вҖ” Glue is only called if an exception is raised |
| 4 | Dependency issues inside container | Listed all packages in `requirements.txt` and used `pip install -r` in Dockerfile |
